# Introduction to BusWork API

Welcome to the BusWork API documentation. This API provides a comprehensive set of endpoints to interact with the BusWork system, allowing you to manage orders, products, users, and pricing rules.

## Base URL

All API requests should be made to:

```
https://buswork.com.ar/api
```

## Authentication

Most endpoints require authentication using an API key. Include your API key in the `api-key` header of your requests.

```http
api-key: YOUR_API_KEY_HERE
```

## Response Format

All responses are returned in JSON format.

## Rate Limiting

The API implements rate limiting to ensure fair usage. Please refer to the specific endpoint documentation for rate limit details.

## Error Handling

In case of an error, the API will return an appropriate HTTP status code along with a JSON response containing more details about the error.

Example error response:

```json
{
  "error": "Invalid API key"
}
```

## Endpoints

The API is organized into the following main sections:

- Authentication
- Orders
- Products
- Users
- Pricing Rules

Each section has its own set of endpoints. Please refer to the individual sections for detailed information on available endpoints and their usage.
